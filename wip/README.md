# `wip/` — Work In Progress

Anything in this directory is **experimental**. It may:
- Break without notice
- Change shape between commits
- Be deleted entirely once it's promoted, retired, or proven dead

**Production code in `src/`, `tools/`, and `tests/` should NOT import from `wip/` unless wrapped in a `try { require(...) } catch {}` graceful-degradation pattern** (see `src/core/crawler/multimodal/MultiModalCrawlOrchestrator.js` for an example with `wip/labs/analysis-observable/`).

## Lifecycle

| State | Location | What it means |
|---|---|---|
| Active production | `src/`, `tools/`, `tests/` | Owned, tested, documented |
| Experimental | `wip/` | Tinker freely; no stability promise |
| Promoted | `solid/` | Curated stable artifact (see `solid/README.md`) |
| Archived | `docs/archives/`, git history | Historically interesting, no longer current |

## Promoting something out of `wip/`

1. Decide destination: `src/<subsystem>/`, `tools/<category>/`, or `solid/`.
2. Update all consumers (`node tools/dev/js-scan.js --what-imports <wip-path> --json`).
3. Add tests + docs (see `solid/README.md` checklist).
4. Move with `Move-Item`, fix imports, update `docs/INDEX.md`.
5. Note the promotion in `docs/decisions/`.

## Deleting from `wip/`

Free to delete anything here without ceremony, **provided** `js-scan --what-imports` shows zero references from `src/`, `tools/`, `tests/`, `checks/`, `scripts/`, or `package.json`.
