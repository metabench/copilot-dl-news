# Naming & Conventions

- Use kebab-case for Markdown filenames and directories (`docs/reference/build_process.md`).
- Use camelCase for JavaScript variables and functions unless project standards specify otherwise.
- Prefix feature flags and config toggles with domain-specific namespaces (e.g., `advancedPlanningSuite`).
