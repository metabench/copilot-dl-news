# Working Notes – Remote Crawler Application Review

- 2026-03-08 — Session created to review the repo’s existing remote crawler capability as a deployable/operable remote application rather than proposing a greenfield design.
- 2026-03-08 — Initial inventory shows three remote crawler generations in `deploy/remote-crawler/`, `deploy/remote-crawler-v2/`, and `deploy/remote-crawler-v3/`, plus local operator tooling under `tools/crawl/`.
- 2026-03-08 — Early doc/code drift detected: `tools/crawl/AGENT.md` references multiple fleet/v4 tools and `src/v4/`, but `src/v4/` is absent from the current worktree and some referenced `tools/crawl/*.js` files are not present locally.
- 2026-03-08 — `deploy/remote-crawler/server.js` is a real single-domain remote crawler server with start/stop/seed/status/errors/export/intelligence/log APIs plus a watchdog restart loop, but it does not boot in this worktree because `deploy/remote-crawler/lib/crawl-worker.js` requires missing `domain-intelligence` and `self-healing` modules.
- 2026-03-08 — `deploy/remote-crawler-v2/multi-domain-server.js` is the strongest operator-facing design in the repo: multi-domain API server, PM2-backed orchestration, gzip export, replay export, hash-manifest sync, SSE, content inspection, logs/errors, DB reset, idle shutdown, and dynamic domain management. It also does not boot in this worktree because the required `lib/schema.js` and several other helper modules are absent.
- 2026-03-08 — `deploy/remote-crawler-v3/` appears incomplete in the current worktree: only `data/`, `node_modules/`, cache files, and `package-lock.json` were present during inspection, with no clear server entrypoint.
- 2026-03-08 — `tools/crawl/crawl-remote.js` remains a usable thin operator CLI front door for the multi-domain server (`status`, `health`, `start`, `bounded`, `stop`, `run`, `pull`, `sync`, `errors`, `content`), and `tools/crawl/lib/crawl-remote-bounded.js` plus `tests/tools/crawl-remote-bounded.test.js` show focused hardening around bounded remote orchestration.
- 2026-03-08 — The current local admin/dashboard story is partial rather than complete for remote operations: `src/ui/server/unifiedApp/server.js` still registers `remote-crawl-admin`, but `src/ui/server/remoteCrawlAdmin/server.js` is missing. `src/ui/server/crawlerMonitor/server.js` exists, but it monitors local coordinator metrics rather than serving as a remote crawler control plane.
- 2026-03-08 — The generic Docker deployment stack (`deploy/docker-compose.yml`, `deploy/Dockerfile`, `deploy/config/production.json`) looks like a separate production crawler architecture based on Postgres/Redis plus dashboard containers. It is not obviously wired to the remote-crawler-v2 multi-domain server path, so deployment guidance is currently split across two different systems.
- 2026-03-08 — Historical notes in `docs/sessions/2026-01-11-crawler-to-country-hubs/WORKING_NOTES.md` show that `deploy/remote-crawler/` was previously deployed to Oracle Cloud and operated via API, but those notes are stronger than what the current worktree can boot today.
