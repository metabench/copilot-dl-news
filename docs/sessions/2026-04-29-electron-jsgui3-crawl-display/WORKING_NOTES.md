# Working Notes: Electron jsgui3 Crawl Display

- 2026-04-29: Session opened after UI tooling review. Goal is an end-to-end validation loop: small crawl -> jsgui3 unified UI -> Electron/screenshot display -> inspect -> fix.
- 2026-04-29: Electron unified smoke initially failed with a native-module ABI mismatch for `better-sqlite3`; root cause was `src/ui/electron/unifiedApp/main.js` requiring the unified server inside Electron's bundled Node. Updated the launcher to spawn the unified jsgui3 server with system Node and use Electron only as the display shell; also added `--screenshot` and `--url-path` support for visual validation.
- 2026-04-29: After split-process fix, Electron reached the system Node server but full unified startup crashed when optional Design Studio assets were absent from the slimmed repo. Hardened unified router mounting so synchronous sub-app factory failures are logged and skipped instead of taking down the crawler UI shell.
- 2026-04-29: During visual prep, found the unified Downloads panel had a Recent Downloads region that never populated. Added `/api/downloads/recent`, wired the panel activator to fill it, and added compact WLILO-aligned row styling so the fresh crawl is visible in screenshots.
- 2026-04-29: Electron `capturePage()` produced empty PNGs in this environment even though the shell smoke path works. Added `scripts/ui/capture-unified-crawl-display.js` to capture and inspect the same unified jsgui3 crawl routes through Puppeteer with deterministic screenshot artifacts and JSON metrics.
- 2026-04-29: Screenshot inspection caught `Unexpected token '&'` in the Crawl Status iframe. The inline client script had been escaped by jsgui3 when added as a plain string; switched it to `String_Control` so the browser executes the script and replaces loading placeholders before capture.
- 2026-04-29: Screenshot inspection after the script fix still showed the Crawl Status operation status as `Loading operations…`; updated the seeded client state to `Ready to start a crawl.` and added a 204 favicon route to remove screenshot-run console noise.
- 2026-04-29: Electron route smoke could close immediately after the outer shell load and reset the Crawl Status iframe request. Added `--smoke-ready-delay-ms` (default 1000ms) so route smokes allow embedded app requests to settle before shutdown.
- 2026-04-29: Added `npm run ui:crawl-display-screenshots` for repeatable crawl-display screenshot capture and documented it in `src/ui/README.md`.
- 2026-04-29: Confirmed the Downloads stats API uses snake_case keys from `downloadEvidence.getGlobalStats`; updated the Downloads activator to handle both snake_case and camelCase shapes and format byte totals consistently.
- 2026-04-29: Tightened `scripts/ui/capture-unified-crawl-display.js` so reports include Downloads stat-card text and fail on non-console browser events, horizontal overflow, or lingering iframe loading text.
